  <behavior>
    CSV export (test_csv.py):
    - export_csv writes a file that starts with UTF-8 BOM bytes (EF BB BF)
    - export_csv writes header row with columns in exact order: region, category, name, address, phone, website, description, source_url, scraped_at
    - export_csv writes one data row per record with correct field mapping
    - export_csv handles None fields as empty strings in CSV
    - export_csv handles fields containing commas, quotes, and newlines (RFC 4180 quoting)
    - export_csv serializes scraped_at datetime as ISO format string
    - export_csv output has no extra blank lines between rows

    JSON export (test_json.py):
    - export_json produces valid JSON parseable by json.loads
    - export_json has top-level keys "metadata" and "regions"
    - export_json metadata contains: scraped_at (ISO string), target_url, total_records (int), schema_version ("1.0")
    - export_json groups records by region, then by category within each region
    - export_json each region has "name" and "categories" keys
    - export_json each category has "name" and "records" keys
    - export_json records do NOT include region/category fields (they are structural)
    - export_json is pretty-printed with 2-space indent
    - export_json uses UTF-8 encoding with ensure_ascii=False
  </behavior>
  <action>
    1. Create `src/export/csv_export.py` with `export_csv(records: list[DirectoryRecord], output_path: Path) -> None`:
       - Open with `encoding='utf-8-sig'` and `newline=''`
       - Use `csv.DictWriter` with fieldnames = ["region", "category", "name", "address", "phone", "website", "description", "source_url", "scraped_at"]
       - Use `extrasaction='ignore'` to handle any extra fields in model_dump
       - Write header, then iterate records calling `writer.writerow(record.model_dump(mode='json'))`
       - `model_dump(mode='json')` automatically converts datetime to ISO string and None to None (DictWriter handles None as empty)

    2. Create `src/export/json_export.py` with `export_json(records: list[DirectoryRecord], output_path: Path, base_url: str) -> None`:
       - Sort records by (region, category) using operator.attrgetter
       - Group by region using itertools.groupby, then within each region group by category
       - Build nested structure: {"metadata": {...}, "regions": [{"name": ..., "categories": [{"name": ..., "records": [...]}]}]}
       - Each record serialized with `model_dump(mode='json', exclude={"region", "category"})`
       - metadata: scraped_at = datetime.now(timezone.utc).isoformat(), target_url = base_url, total_records = len(records), schema_version = "1.0"
       - Write with `json.dump(output, f, indent=2, ensure_ascii=False)`
       - NOTE: itertools.groupby requires sorted input. The sorted() call MUST come before groupby.

    3. Write RED tests first using pytest's tmp_path fixture for output files. Read back the files and verify content.
  </action>
