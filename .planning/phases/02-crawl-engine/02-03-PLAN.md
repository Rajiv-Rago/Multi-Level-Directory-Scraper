src/scraper/pagination.py:

PaginationConfig dataclass:
- type: Literal["next_page", "load_more", "infinite_scroll"]
- selector: str (CSS selector for next link / load more button / item count)
- max_pages: int = 100
- max_items: int | None = None (for infinite_scroll)
- content_selector: str | None = None (for detecting new content after load_more)

PaginationHandler class:
- __init__(self, fetcher): stores reference to the fetcher

- async paginate(self, initial_html: str, pagination_config: PaginationConfig | None, base_url: str) -> list[str]:
  - If no pagination_config or type is None: return [initial_html]
  - Dispatch to _next_page, _load_more, or _infinite_scroll based on type
  - Return list of HTML strings (one per page)

- async _next_page(self, initial_html, config, base_url) -> list[str]:
  - Parse HTML, find next link selector
  - Follow chain: fetch next URL, find next link, repeat
  - Stop when: no next link found OR max_pages reached
  - Return all HTML pages collected

- async _load_more(self, config, base_url) -> list[str]:
  - This needs Playwright page access (can't work with just HTML string)
  - Fetcher must expose page for load_more: add a fetch_with_page method to PlaywrightFetcher that returns (html, page) tuple, caller closes page
  - Click load_more selector, wait for content_selector, repeat
  - Return final accumulated HTML as single string in list

- async _infinite_scroll(self, config, base_url) -> list[str]:
  - Similar to load_more: needs Playwright page access
  - Scroll, count items matching selector, repeat until no change or max_items
  - Return final HTML as single string in list

src/scraper/orchestrator.py:

LevelConfig dataclass (or use existing from Phase 1 SiteConfig):
- depth: int
- name: str
- link_selector: str | None (None for detail pages)
- context_selector: str | None
- wait_selector: str | None
- renderer: str = "browser"
- pagination: PaginationConfig | None
- fields: list[dict] | None (only for detail pages)
- is_detail_page: bool = False

CrawlOrchestrator class:
- __init__(self, config, fetcher, extractor, frontier, pagination_handler, delay_fn):
  - config: site configuration with levels list
  - fetcher: Fetcher instance (wrapped with retry)
  - extractor: Extractor instance
  - frontier: URLFrontier instance
  - pagination_handler: PaginationHandler instance
  - delay_fn: async callable for polite delays (from Phase 1, or simple asyncio.sleep fallback)

- async crawl(self) -> list[dict]:
  - Seed frontier with base_url at depth 0
  - For each level in config.levels (ordered by depth):
    - While frontier.has_pending(level.depth):
      - item = frontier.pop(level.depth)
      - html = await self._fetch(item.url, level)
      - If html is None: log skip, continue
      - pages = await pagination_handler.paginate(html, level.pagination, item.url)
      - For each page_html in pages:
        - If level.is_detail_page:
          - record = extractor.extract_record(page_html, level.fields, item.ancestors)
          - record["_source_url"] = item.url
          - records.append(record)
        - Else:
          - links = extractor.extract_links(page_html, level.link_selector)
          - context_label = extractor.extract_context(page_html, level.context_selector)
          - ancestors = item.ancestors + [{"level": level.name, "label": context_label, "url": item.url}]
          - For each link: frontier.add(urljoin(item.url, link), level.depth + 1, ancestors)
      - await delay_fn()
  - Log summary: total records, total URLs visited, per-level counts
  - Return records

- async _fetch(self, url, level) -> str | None:
  - Choose fetcher based on level.renderer
  - Call fetch with level.wait_selector and timeout from config
  - Return HTML or None on failure

The orchestrator handles NAV-01 (discover region/category links), NAV-02 (discover entity links), NAV-03 (pagination via handler), NAV-05 (log failures and continue), and EXT-03 (ancestor context).
