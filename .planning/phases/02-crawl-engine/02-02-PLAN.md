src/scraper/extractor.py:

Extractor class:
- extract_field(soup: BeautifulSoup, selectors: list[str], attribute: str | None = None) -> str | None
  - For each selector in order: soup.select_one(selector)
  - If element found: return element.get(attribute) if attribute else element.get_text(strip=True)
  - If no selector matches: return None

- extract_record(html: str, fields_config: list[dict], ancestors: list[dict]) -> dict
  - Parse HTML with BeautifulSoup(html, "lxml")
  - For each field in fields_config: call extract_field with field's selectors and attribute
  - Build record dict: {field.name: extracted_value for each field}
  - Add ancestor context: record["_region"] = first ancestor with level containing "region" (or level at depth 1), record["_category"] = similar for category
  - Add record["_source_url"] = will be set by orchestrator (not extractor's job)
  - Return record

- extract_links(html: str, link_selector: str) -> list[str]
  - Parse HTML, select all matching elements, return [a.get("href") for a if a.get("href")]

- extract_context(html: str, context_selector: str) -> str | None
  - Parse HTML, select_one, return text or None

src/scraper/retry.py:

Create a resilient_fetch async function (or decorator factory):

Option A -- decorator approach:
```python
def with_retry(fetch_fn):
    @retry(
        retry=retry_if_exception_type(RateLimitError),
        wait=wait_exponential(multiplier=1, min=5, max=60),
        stop=stop_after_attempt(3),
        before_sleep=before_sleep_log(logger, logging.WARNING),
        reraise=True
    )
    @retry(
        retry=retry_if_exception_type(ServerError),
        wait=wait_fixed(10),
        stop=stop_after_attempt(2),
        before_sleep=before_sleep_log(logger, logging.WARNING),
        reraise=True
    )
    async def wrapper(*args, **kwargs):
        return await fetch_fn(*args, **kwargs)
    return wrapper
```

Note: Stacked tenacity decorators work -- inner decorator handles ServerError (2 attempts), outer handles RateLimitError (3 attempts with exp backoff). Test to confirm the composition behaves correctly; if stacking is problematic, combine into a single decorator with retry_if_exception_type(RateLimitError) | retry_if_exception_type(ServerError) and a custom wait function that checks the exception type.

Alternative: single decorator with custom wait callback:
```python
def custom_wait(retry_state):
    exc = retry_state.outcome.exception()
    if isinstance(exc, RateLimitError):
        return wait_exponential(multiplier=1, min=5, max=60)(retry_state)
    elif isinstance(exc, ServerError):
        return 10
    return 0
```

Choose whichever approach passes the tests cleanly. The important thing is the behavior matches RES-01 and RES-02 specs.

Use structlog for the before_sleep logging.
