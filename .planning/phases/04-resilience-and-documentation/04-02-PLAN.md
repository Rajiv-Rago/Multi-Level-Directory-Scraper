Write README.md with these sections (in order):

**1. Title and badges**
- Project name: Multi-Level Directory Scraper
- Python version badge, license badge

**2. Problem Statement**
- What: A config-driven scraper for hierarchical, JS-rendered directory websites
- Why it's non-trivial: multi-level navigation (regions -> listings -> details), JavaScript rendering required, pagination handling, data quality challenges
- What it produces: clean, validated CSV and JSON datasets

**3. Key Features**
- Bullet list: config-driven, multi-level navigation, JS rendering (Playwright), pagination handling, data validation, deduplication, checkpoint/resume, structured logging

**4. Quick Start**
- 3 steps: clone, install (`pip install -r requirements.txt` + `playwright install`), run (`python -m scraper --config config.yaml`)
- Must be achievable in under 5 minutes

**5. Architecture / Approach**
- Brief description of the crawl pipeline: config loading -> level-by-level navigation -> JS rendering -> extraction -> validation -> output
- Key decisions: why Playwright (JS rendering), why config-driven (reusable across sites), why BeautifulSoup (fast HTML parsing after render)
- Simple text diagram of the pipeline flow (not a complex ASCII art — just a clean flow)

**6. Data Quality**
- Validation pipeline: phone normalization, URL resolution, whitespace cleanup, deduplication
- Metrics achieved (from validation report or realistic estimates): field completeness %, duplicates detected, etc.

**7. Resilience**
- How it handles failures: HTTP retry with exponential backoff, timeout recovery, polite crawling
- Checkpoint/resume: saves state on Ctrl+C, resumes without re-scraping
- Embed real log snippets from docs/samples/log_*.txt showing:
  - Retry in action
  - Checkpoint save on interruption
  - Resume from checkpoint

**8. Sample Output**
- Truncated CSV (embed first 5-10 rows as a markdown table)
- JSON snippet (embed as code block)
- Note about validation report

**9. Configuration**
- Annotated example config.yaml showing: base_url, per-level selectors, pagination rules, field mappings, delay settings

**10. Setup Instructions**
- Prerequisites: Python 3.10+, system dependencies for Playwright
- Step-by-step install
- Verify installation command
- Run with sample config
- Expected output

**Tone:** Technical but accessible. Lead with results, show don't tell. Written for senior engineers and hiring managers.
