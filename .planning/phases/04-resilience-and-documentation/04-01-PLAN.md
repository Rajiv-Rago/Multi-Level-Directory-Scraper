**CheckpointManager class:**
- `__init__(self, output_dir: Path, config_hash: str)` — stores output dir and config hash
- `checkpoint_path` property — returns `output_dir / '.checkpoint.json'`
- `save(self, state: dict)` — atomic write: tempfile.mkstemp in same dir, json.dump, os.replace. - `cleanup(self)` — delete checkpoint file if it exists.
- `exists` property — returns True if checkpoint file exists.

**Module-level convenience functions:**
- `config_hash(config: dict) -> str` — hash base_url + selectors fields with sha256, return first 16 hex chars.

All writes must be atomic (tempfile + os.replace). Use logging module for warnings.

<task type="auto">
  <name>Task 4: Implement SignalHandler with cooperative shutdown and partial result persistence</name>
  <files>src/signals.py, src/cli.py</files>
  <action>
Create src/signals.py implementing:

**SignalHandler class:**
- `__init__(self, checkpoint_manager: CheckpointManager, get_state: Callable, flush_results: Callable)` — stores references to checkpoint manager and callbacks for getting current crawl state and flushing partial results.
- `shutdown_requested: bool` — flag checked by crawl loop.
- `shutdown_count: int` — tracks number of signals received.
- `register(self)` — register handlers for SIGINT and SIGTERM via signal.signal().
- `_handle_signal(self, signum, frame)` — increment shutdown_count, set shutdown_requested=True. If shutdown_count == 1: log "Received signal, finishing current page and saving state..." If shutdown_count >= 2: log "Force shutdown — saving emergency state...", call self._emergency_save(), sys.exit(1).
- `save_state(self)` — called by crawl loop after detecting shutdown_requested. Calls get_state() to get current state dict, calls checkpoint_manager.save(state), calls flush_results() to write partial output. Logs "Checkpoint saved: N pages visited, M remaining" and "Partial results saved: K records".
- `_emergency_save(self)` — same as save_state but wrapped in try/except (best-effort on second signal).

**CLI integration (src/cli.py):**
- Add `--resume` flag to CLI argument parser.
- Add `--force` flag (for resuming with mismatched config).
- On startup: if --resume, call checkpoint_manager.load(force=args.force). If checkpoint exists but --resume not passed, log hint: "Checkpoint found. Use --resume to continue from where you left off."

**Periodic checkpointing integration point:**
- Add a `should_checkpoint(self, pages_since_last: int, interval: int = 50) -> bool` method that returns True when pages_since_last >= interval. The actual save is done by the crawl loop calling save_state().
