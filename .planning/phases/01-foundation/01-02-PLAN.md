**Implementation** (`src/scraper/politeness.py`):

`PolitenessController` class:
- `__init__(self, config: ScrapeConfig, logger)` — stores config, logger
- `async def initialize(self)` — fetches robots.txt via httpx from `{base_url}/robots.txt`, parses with `urllib.robotparser.RobotFileParser`. Also extracts Crawl-delay manually by reading raw text lines and matching `Crawl-delay: {value}` (since stdlib robotparser doesn't support Crawl-delay). Computes effective_delay_min = max(config.request_delay.min, crawl_delay or 0). On fetch failure (any exception), logs warning and sets allow_all=True.
- `def is_allowed(self, url: str) -> bool` — checks URL against robots.txt rules. If allow_all, returns True. Logs skipped URLs at WARNING level.
- `async def wait(self)` — sleeps for `random.uniform(self.effective_delay_min, config.request_delay.max)`. Logs delay at DEBUG level.
- `@property def effective_delay_min(self) -> float` — computed min delay considering robots.txt

User-agent for robots.txt: "multi-level-directory-scraper" (matches the project name).

**Implementation** (update `src/scraper/cli.py`):

Add a `run_dry_run(config: ScrapeConfig, logger)` function:
1. Initialize PolitenessController and call `await controller.initialize()`
2. For each configured level:
   - Determine URL to fetch: level 0 uses `config.site.base_url`, deeper levels log "would follow links from level {n-1}" and skip actual fetching (no crawl engine yet in Phase 1)
   - Fetch the page using `httpx.AsyncClient.get(url)` with the configured user-agent
   - Check `controller.is_allowed(url)` before fetching
   - Parse response HTML with BeautifulSoup (bs4 + lxml)
   - For each field in the level's field config, apply the CSS selector
   - Collect results: field name, selector used, extracted value or "NOT FOUND"
3. Print results table to terminal. Use a simple formatted table (column-aligned with headers: Level, Field, Selector, Value). If rich is available, use `rich.table.Table` for nicer formatting; otherwise fall back to plain text alignment.
4. Print summary: "X of Y fields extracted successfully"
5. Exit 0 if at least one field per level extracted, exit 1 if any level has zero extractions

Update the `main()` CLI function:
- When `--dry-run` is True, call `run_dry_run(config, logger)` instead of the placeholder
- Do NOT set up file logging in dry-run mode (pass log_file=None to setup_logging)
- Wrap in `asyncio.run()` since PolitenessController is async

Note: For deeper levels (depth > 0), dry-run in Phase 1 can only validate selectors if a sample URL is provided. For now, only fetch level 0 (base_url). Log info message for deeper levels: "Level '{name}' (depth {n}): selector validation requires crawl engine (Phase 2)". This is acceptable per the context doc — dry-run validates what it can.
