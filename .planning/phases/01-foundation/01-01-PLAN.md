<task type="auto">
  <name>Task 2: Implement config models and YAML loading</name>
  <files>src/scraper/config.py, configs/example.yaml, tests/conftest.py, tests/test_config.py</files>
  <action>
Create Pydantic v2 models in `src/scraper/config.py`:

1. **Models** (all with `model_config = ConfigDict(frozen=True)`):
   - `RequestDelayConfig`: min (float, default 1.0), max (float, default 3.0). Add `model_validator(mode='after')` ensuring min <= max.
   - `PaginationConfig`: type (Literal["next_button", "load_more", "infinite_scroll", "none"]), selector (str, optional).
   - `FieldMapping`: name (str), selector (str), attribute (str, default "text"), default (str | None, default None).
   - `LevelConfig`: name (str), depth (int >= 0), link_selector (str), pagination (PaginationConfig | None, default None), fields (list[FieldMapping], min 1 item).
   - `SiteConfig`: name (str), base_url (str, validated as URL), output_dir (str, default "./output"), request_delay (RequestDelayConfig, default factory), max_pages (int | None, default None), log_level (Literal["debug", "info", "warning"], default "info").
   - `ScrapeConfig`: site (SiteConfig), levels (list[LevelConfig], min 1 item). Add `model_validator(mode='after')` ensuring level names are unique and depths are sequential starting from 0.

2. **`load_config(path: Path) -> ScrapeConfig`**: Read YAML file with PyYAML, pass to `ScrapeConfig.model_validate()`. Catch `ValidationError`, re-raise with clear message. Catch `FileNotFoundError` and `yaml.YAMLError` with user-friendly messages.

3. **`apply_overrides(config: ScrapeConfig, overrides: dict) -> ScrapeConfig`**: Since models are frozen, create new instances with overridden values using `model_copy(update=...)`. Only apply non-None override values. Handle nested overrides (delay_min/delay_max update the nested RequestDelayConfig).

4. **`configs/example.yaml`**: Annotated example with comments showing all fields, including a 2-level setup (regions -> listings) with example selectors and field mappings.

5. **Tests** — Write tests FIRST (TDD per CLAUDE.md):
   - `tests/conftest.py`: Fixtures for valid config dict, minimal config dict, tmp_path YAML writing helper.
   - `tests/test_config.py`:
     - `test_valid_config_loads` — full config loads without error, all fields accessible
     - `test_minimal_config_defaults` — only required fields, defaults applied correctly
     - `test_missing_required_field_error` — clear error for missing `base_url`
     - `test_invalid_delay_range` — min > max raises validation error
     - `test_duplicate_level_names` — raises validation error
     - `test_config_is_immutable` — assigning to field raises error
     - `test_apply_overrides` — CLI overrides correctly replace config values
     - `test_load_nonexistent_file` — clear FileNotFoundError message
  </action>

  <name>Task 3: Implement CLI and structured logging</name>

  <action>
1. **`src/scraper/logging.py`** — `setup_logging(log_level: str, log_file: Path | None)`:
   - Use `logging.config.dictConfig` for dual output:
     - Console handler: `structlog.dev.ConsoleRenderer(colors=True)` for pretty colored output
     - File handler (if log_file provided): `structlog.processors.JSONRenderer()` for JSON lines
   - Shared processor chain: `add_log_level`, `TimeStamper(fmt="iso")`, `StackInfoRenderer()`, `format_exc_info`, `UnicodeDecoder()`, `ProcessorFormatter.wrap_for_formatter`
   - Configure structlog with `stdlib.BoundLogger` wrapper, `stdlib.LoggerFactory()`
   - Return a bound logger via `structlog.get_logger()`
   - If log_file is None (dry-run mode), skip file handler

2. **`src/scraper/cli.py`** — Typer app with single command:
   ```python
   app = typer.Typer(name="scraper", add_completion=False)

   @app.command()
   def main(
       config_path: Annotated[Path, typer.Argument(help="Path to YAML config file", exists=True)],
       output_dir: Annotated[Optional[str], typer.Option("--output-dir", help="Override output directory")] = None,
       delay_min: Annotated[Optional[float], typer.Option("--delay-min", help="Minimum request delay (seconds)")] = None,
       delay_max: Annotated[Optional[float], typer.Option("--delay-max", help="Maximum request delay (seconds)")] = None,
       max_pages: Annotated[Optional[int], typer.Option("--max-pages", help="Maximum pages per level")] = None,
       log_level: Annotated[Optional[str], typer.Option("--log-level", help="Log level: debug/info/warning")] = None,
       dry_run: Annotated[bool, typer.Option("--dry-run", help="Validate config and test selectors without full crawl")] = False,
   ):
   ```
   - Load config via `load_config(config_path)`
   - Build overrides dict from non-None CLI args
   - Apply overrides via `apply_overrides(config, overrides)`
   - Set up logging via `setup_logging(config.site.log_level, log_file_path)` — log_file is `{output_dir}/scrape.log` unless dry_run
   - Log startup event: `log.info("config_loaded", site=config.site.name, levels=len(config.levels))`
   - If dry_run: placeholder message "Dry-run mode: config validated successfully" (actual dry-run logic in Plan 02)
   - Wrap config loading in try/except: `ValidationError` -> exit code 1 with error message, `Exception` -> exit code 2
   - Exit code 0 on success
